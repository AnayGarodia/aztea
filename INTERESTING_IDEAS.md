# Interesting Ideas

Add raw product ideas here when they are worth saving.

## Git for Agent Actions Outside Code

Coding agents work because code already has branches, diffs, tests, commits, and rollback. Most non-code agent work touches production life directly: email, calendar, CRM, payments, files, docs, support tools, and customer messages.

Idea: give agents a branch of the real world. Let an agent work in a staged version of Gmail, Calendar, Drive, Docs, Sheets, CRM, or commerce tools, then produce a diff that can be auto-committed when safe or reviewed when risky.

The promise: agents can act more autonomously because their actions are staged, inspectable, testable, and undoable.

Possible wedge: Google Workspace branch for agents: shadow outbox, calendar dry-run, Drive snapshots, Docs/Sheets diffs, contact-change diffs, auto-commit rules, and rollback.

## Aztea Agent Market / AgentNet

Aztea already has the bones of agent hiring: discovery, wallet, job lifecycle, receipts, settlement, reputation, disputes, and recourse. The missing piece is not another narrow specialist catalog. The missing piece is a place where agent-to-agent work actually happens.

Idea: turn Aztea from static transaction rails into an active agent labor market. Agents have profiles, addresses, wallets, offers, public work history, and the ability to hire or be hired by other agents. Humans post work or create agents; agents find counterparties, negotiate scope, submit results, and build reputation through completed jobs.

The useful version should focus on real work, not agent social content. The visible product is a market/network of agents completing tasks; the Aztea rails underneath handle money, trust, proof, and recourse.

Possible wedge: public tiny jobs where agents compete or collaborate on research, data cleanup, QA, monitoring, design variants, or code verification, with visible receipts and reputation.
